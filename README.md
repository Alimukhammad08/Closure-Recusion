# Closure и Recursion 

В этом репозитории собраны примеры и краткие заметки по темам **Closure (замыкания)** и **Recursion (рекурсия)** в JavaScript.

## Что такое Closure?

Замыкание — это функция, которая запоминает переменные из своей внешней области видимости даже после того, как внешняя функция завершила работу.

### Пример

```javascript
function createCounter() {
  let count = 0;

  return function () {
    count++;
    return count;
  };
}

const counter = createCounter();

console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3
```

### Как это работает?

После вызова `createCounter()` обычно локальные переменные должны удалиться из памяти. Но внутренняя функция продолжает использовать переменную `count`, поэтому JavaScript сохраняет её.

Таким образом значение `count` не сбрасывается при каждом вызове.

### Где используются замыкания?

* создание счётчиков;
* хранение состояния;
* инкапсуляция данных;
* обработчики событий;
* фабрики функций.

Пример фабрики функций:

```javascript
function multiplyBy(multiplier) {
  return function(number) {
    return number * multiplier;
  };
}

const double = multiplyBy(2);

console.log(double(5)); // 10
```

## Что такое Recursion?

Рекурсия — это способ решения задачи, при котором функция вызывает саму себя.

Для правильной работы рекурсии обязательно нужен **базовый случай** — условие, при котором вызовы прекращаются.

### Пример: факториал

```javascript
function factorial(n) {
  if (n === 1) {
    return 1;
  }

  return n * factorial(n - 1);
}

console.log(factorial(5)); // 120
```

### Как работает рекурсия?

Для `factorial(5)` вычисления происходят так:

```text
5 × factorial(4)
5 × 4 × factorial(3)
5 × 4 × 3 × factorial(2)
5 × 4 × 3 × 2 × factorial(1)
= 120
```

---

### Пример: обратный отсчёт

```javascript
function countdown(n) {
  if (n === 0) {
    console.log("Старт!");
    return;
  }

  console.log(n);
  countdown(n - 1);
}

countdown(5);
```

## Отличия Closure и Recursion

| Closure                                         | Recursion                                                  |
| ----------------------------------------------- | ---------------------------------------------------------- |
| Запоминает переменные внешней области видимости | Функция вызывает саму себя                                 |
| Позволяет хранить состояние                     | Позволяет разбивать задачу на более простые части          |
| Часто используется для инкапсуляции данных      | Часто используется для обхода структур данных и вычислений |

---

## Вывод

Замыкания помогают функциям сохранять доступ к данным из внешней области видимости, а рекурсия позволяет решать сложные задачи через последовательное разбиение на более простые.

Обе темы являются важной частью JavaScript и часто используются как в реальных проектах, так и на технических собеседованиях.
